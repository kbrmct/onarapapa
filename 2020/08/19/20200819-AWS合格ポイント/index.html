AWSクラウドプラクティショナー合格への傾向と対策

ども、おならパパです！

先日、AWSクラウドプラクティショナー試験に一発合格してきました。IT業界２０年の大ベテランのおっさんが落ちるわけにはいきません。

40超えたおっさんが、ベンダー系の資格受けるのか！という突込みは受け付けません。

大企業の一員である以上、若手の見本になる必要もあるのです！

ここでは、AWSクラウドプラクティショナー試験に合格するための傾向と対策を解説します。

これからのITはクラウド大前提！

AWSクラウドプラクティショナー試験を受ける意義について解説いたします。

システムエンジニアとして仕事をする場合、大きくフロントエンド、バックエンドがあります。

フロントエンドは、パソコン、スマホなどで、ユーザが操作するインターフェースに関わる部分。

バックエンドは、文字通り、システムの裏方部分で、ユーザの操作に合わせてデータを引っ張ってきたり、保存したり、管理したりする部分です。

クラウドがない時代、バックエンドを担当するエンジニアは、

  • 想定されるユーザ数、操作から、トラヒック量を計算

  • 処理を捌ける、アーキテクチャを設計し、必要なサーバ数、ネットワーク機器を特定

  • 実際に機器を調達し、設置

  • 必要ソフトウェアのインストール

  • 必要な機能をコーディングし、実装

という作業が必要でした。

クラウドが利用できる現在は、

  • 必要なソフトウェアが入ったサーバイメージを選択し、必要な機能をコーディングし実装

  • 必要最低限のサイズの仮想サーバ上で動かし、ユーザを獲得するにつれスケールアップ

と、システムの作り方がガラリと変わります。

今までマンパワーが必要だったところがすべてマウスのクリックだけでできてしまいます。

新しいサービスはクラウドで動かすことを前提とし、限られた必須の機能を開発し、サービス開始し、それを大きく育てる流れです。

クラウドの発祥の地、米国では、米国連邦政府自ら、Federal Cloud Computing Strategyを打ち出し、この先進的なIT技術の適用を初めています。マイクロソフトは、米国防総省のシステムのクラウド移行を、１兆円で受注しています。

日本政府も、クラウド・バイ・デフォルト原則として、各府省で政府情報システムを整備するにあたって、クラウドサービスの利用を第一候補として検討するという方針を打ち出しました。日本政府専用のクラウド環境である政府共通プラットホームも、AWSによって構築されることになりました。

今後、クラウド技術を持つバックエンドのエンジニアはますます重要となるでしょう。しかし、クラウドのエンジニアはまだまだ不足していると言われています。

開発者視点でクラウドを理解をしている人材は、引く手あまた。これからも貴重な戦力となるでしょう。

どのクラウドがいいの？AWS ? Azure ?

クラウドを提供する業界１位は、amazonのAWSですが、それだけではありません。

マイクロソフトが提供するクラウドAzureもあります。（業界２位）

Googleが提供するGoogle Cloudもあります。（業界３位）

この３つを足すと、シェア55%となります。

この３つのどれかに携わっておけば、仕事がなくなることは当面はないでしょう。かといって、３社のクラウド技術を一人で使いこなすのは、大変ですから、どれか１社のプロになったほうがよいでしょう。

私は、業界1位のamazon AWSで、クラウドの基本を体得するため、試験を受けました。

世界の半分を米国の３社が牛耳っているのは、日本人として悲しい状況ではありますが、長いものにはグルグル巻き、世界の大きな潮流に身をのっけて、生きていくしかありません。

ちなみに業界４位はAlibabaの中国製。まこちらを喜んで使う人はいないので、無視して良いと思います。

AWS クラウドプラクティショナーの試験内容

試験問題数と合格ライン

試験問題数は65問。合格ラインは7割正解、つまり46問正解が必要と言われております。
1000点満点で、700以上で合格です。
受験後、点数はでますが、どこ間違えたかは、わかりません。 ちなみに私は、800点で合格です。

試験対策の教材

私が実施したのは、以下３つです。

  • 参考書

AWS認定資格試験テキスト AWS認定 クラウドプラクティショナーを参考書として利用し、全体概要を理解しました。各章の問題例があるので、そこで理解度を確認できます。

  • AWSが提供するオンラインの教材

アマゾンはより多くの方にAWSを利用してもらうため、充実したコンテンツを提供しております。

AWS認定試験に備えるにて、事前学習教材や、問題例にアクセスできます。

  • 一日研修

AWSの集中研修です。「ここ試験で出るとこ！」とやってくれますが、かなりの詰め込み学習なので、大変です。

実際にAWSを触ってみて、webサーバが立ち上がりますね、という体験付きでした。触ったことある人にとっては、そうでしょうね、ですし、初めての人にとっては、クリックだけでできてしまうので、へぇ、という感じに映っているようです。

研修を受けなくても、対策本とAWSが提供する教材を使えば、合格は可能だと思います。

役立つ前提となる知識

システム開発経験を通して、サーバ、ネットワーク、データベース、セキュリティをやってきた人はかなり優位になります。

結局、物理的に組むものを、AWSのクラウド空間に置いたらどうなるのか？という観点で考える必要がありますので、いきなりクラウドに飛びついても、腑に落ちない部分は多いでしょう。

実際、システム開発経験の少ない若い社員は、何を言っているかわからない、と面食らっていました。

基本情報技術者レベルの知識と、システム開発経験はあったほうが良いでしょう。

試験問題の傾向

似た問題が2,3回出題されました。間違えて覚えていたら全滅する可能性があるので、記憶があいまいな場合は、答えを分散させてリスクヘッジしましょう。

私が受験したときは、以下に関する事項が、問題文を変えて数回登場しました。

  • 一年のうちに1日だけ24時間動かすシステムがあり、その時は、どのプランが良い？

    スポットインスタンス、オンデマンドインスタンスが選択肢にあり、うっかり、スポットインスタンスを選びそうですが、オンデマンドインスタンスが正解。

  • AWSオーガニゼーション利用時のメリットに関する問題

    組織で一元管理できたら、どのようなメリットがあるのか、想像する必要がある。

  • コンプライアンスに関する問題

    AWSが高いセキュリティを保証するために、どのような法律に準拠しているのか、理解する必要があります。また、利用者とAWSとの責任共有モデルについても、理解を求められます。

  • コスト管理に利用するツールに関する問題

    AWS budget , cost explorer の使う目的を理解すること。

  • TCOを計算する上で考えるべきことは何か。

    もはやAWSの話ではありませんが、TCOはどういう範囲で計算するべきか、考え方を理解し、AWSはTCO削減に非常に有効な環境である、という答えを選びましょう。

試験環境

私は東京駅近くの試験センターで受講しました。

PCが大量に並んでいる教室で、席をもらい、耳栓替わりの大きなヘッドホンを渡され、黙々と解いていきます。

米国の資格試験を日本語にしていますので、時々日本語が変な時があります。その時は、英語に切り替えて、問題の意図を確認するとよいです。

まとめ

  • AWSの合格の秘訣は、サーバ、ネットワーク、データベース、セキュリティの基本的な部分を理解していること。基本情報技術者試験レベルの知識があったほうがよい。
  • AWSの強みをよく理解し、設問に答えましょう。
  • これからしばらくは、クラウドが大前提のIT業界となるでしょう。クラウドが使える／使えないで、食い扶持が変わってくるでしょう。
